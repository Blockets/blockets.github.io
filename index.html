Blockets SplitPayments API

Base URL: /api/v1, Version: 0.1.0

The Blockets Split Payments API is an interface that a ticket-selling service can dispatch to, to handle the actual payments of these tickets.

As soon as a payment has happened, the money of this payment is then automatically split and divided between the different parties that ought to obtain money from it.

Default response content-types: application/json
Schemes: https

GET /unreservedTicketsCount

Lightweight call to check approx. how many tickets are still not reserved for a sale right now. This number can be used to e.g. in an interface restrict what the maximum amount of tickets is someone can select. Do note that because pending ticket reservations (monetary transactions) can succeed/fail/time out at any time, this number is only an indication and no guarantee that that number of tickets still is available when someone actually goes ahead to buy them.

Schema definitions

event_identifier: #/parameters/event_identifier

description: string
event_starts_timestamp: string (datetime)
event_ends_timestamp: string (datetime)

No ticket types can be made whose end_selling_timestamp is after this.

FixedPaymentParties: object[]

FixedPaymentParty

FixedPaymentParty: object

name: string
fixed_amount: integer

The amount that will go to this party when a payment occurs, in eurocent.

iban: string

Should be a valid International Bank Account Number. TODO Validate if this is all information that Mollie needs to execute split payments.

FractionPaymentParties: object[]

FractionPaymentParty

FractionPaymentParty: object

name: string
parts: integer

The parts (numerator) that are for this party. The 'whole' (i.e. the denominator) is determined by adding together all the parts in the fraction_payment_parties array.

iban: string

Should be a valid International Bank Account Number. TODO Validate if this is all information that Mollie needs to execute split payments.

TicketType: object

ticket_type_identifier: #/parameters/ticket_type_identifier
name: string
description: string
fixed_payment_parties: FixedPaymentParties
fraction_payment_parties: FractionPaymentParties
price: integer

Price in eurocent. An error will be raised if this price is lower than the amount of the fixed_amounts in the fixed_payment_parties array.

end_selling_timestamp: string (datetime)

After this, no more tickets of this ticket type can be sold.

start_selling_timestamp: string (datetime)

Before this, no tickets of this ticket type can be sold.
